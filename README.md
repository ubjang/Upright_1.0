# 청렴 바이브

내부 직원용 **청렴 인식** 웹앱입니다. 공공기관 **청렴도 평가**에서 다루는 영역을 바탕으로, 월 **1일·15일(한국 시간)** 에 가볍게 체크·퀴즈에 참여할 수 있습니다.  
**신고·제보 기능은 포함하지 않습니다.**

## 스택

- Next.js (App Router), Tailwind CSS v4
- Supabase (Auth + Postgres, **RLS**로 본인 데이터만 조회)

## 설정

1. Supabase 프로젝트를 만듭니다.
2. **Authentication → Providers → Anonymous** 에서 익명 로그인을 켭니다. (개발용; 이후 매직 링크 등으로 교체 가능)
3. SQL 에디터 또는 CLI로 [`supabase/migrations/001_init_rls.sql`](supabase/migrations/001_init_rls.sql)을 실행합니다.
4. `.env.local.example`을 복사해 `.env.local`을 만들고 URL·Anon 키를 넣습니다.

```bash
cp .env.local.example .env.local
npm install
npm run dev
```

브라우저에서 [http://localhost:3000](http://localhost:3000) 을 엽니다.

### 문제 해결

- **`GET / 404` / 화면이 안 뜸**: 개발 서버를 끈 뒤 `.next` 폴더를 삭제하고 `npm run dev`를 다시 실행해 보세요.
- **첫 컴파일이 매우 느림**: 프로젝트가 네트워크 드라이브(D: 등)에 있으면 Next가 경고를 냅니다. 가능하면 로컬 디스크로 옮기면 빨라집니다.
- **포트 충돌**: `3000`이 사용 중이면 Next가 다른 포트를 쓰거나, 기존 프로세스를 종료한 뒤 다시 실행하세요.

## 참여 흐름

- **홈**: 누적 참여 횟수에 따라 “청렴 새싹” 단계가 바뀌고, 이번 달 체크·퀴즈 여부를 볼 수 있어요.
- **체크**: 청렴도 평가 축별 실천 문구 중 **서로 다른 축 3개**를 골라 제출해요.
- **퀴즈**: 사례형 **OX·선택형** 2문제를 풀고, 해설을 본 뒤 한 번에 제출해요.

첫 방문 시 홈만 보면 세션이 없어 카운트가 0일 수 있어요. **체크** 또는 **퀴즈** 페이지에 들어가면 익명 로그인 후 기록이 쌓입니다.

## 배포·운영 시 참고

- 익명 계정은 브라우저·기기별로 달라질 수 있습니다. 실서비스에서는 기관 메일 **매직 링크** 등으로 바꾸는 것을 권장합니다.
- 기관 실제 **청렴도 평가 지표**에 맞춰 `lib/constants/integrityAxes.ts`, `checkPhrases.ts`, `quizBank.ts` 문구를 조정하세요.
